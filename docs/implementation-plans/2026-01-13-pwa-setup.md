# PWA Setup for My Recipe Journal

Add Progressive Web App (PWA) capabilities to enable "Add to Home Screen" functionality on mobile devices.

## Proposed Changes

### Public Assets

#### [NEW] [manifest.json](file:///c:/Users/zheny/Downloads/Dev/my-recipe-journal/public/manifest.json)

Web App Manifest defining:

- App name and short name
- Theme color (matching app's color scheme)
- Background color
- Display mode: `standalone` (fullscreen app experience)
- Start URL
- Icon references (192x192 and 512x512)

#### [NEW] [icon-192.png](file:///c:/Users/zheny/Downloads/Dev/my-recipe-journal/public/icon-192.png)

192x192 app icon (generated from existing goat-chef illustration)

#### [NEW] [icon-512.png](file:///c:/Users/zheny/Downloads/Dev/my-recipe-journal/public/icon-512.png)

512x512 app icon (generated from existing goat-chef illustration)

### Service Worker (Offline Support)

For Next.js, we'll use `next-pwa` to automatically generate a service worker with caching strategies.

#### [MODIFY] [package.json](file:///c:/Users/zheny/Downloads/Dev/my-recipe-journal/package.json)

Add dependency: `next-pwa`

#### [MODIFY] [next.config.mjs](file:///c:/Users/zheny/Downloads/Dev/my-recipe-journal/next.config.mjs)

Configure `next-pwa` with:

- Runtime caching for API routes
- Static asset caching
- Disabled in development mode

#### [NEW] [.gitignore additions](file:///c:/Users/zheny/Downloads/Dev/my-recipe-journal/.gitignore)

Add generated service worker files: `public/sw.js`, `public/workbox-*.js`
